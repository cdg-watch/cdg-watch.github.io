# SNS運用エージェント指針(X / 旧Twitter)

運用アカウント: **@cdg_watch**(2026-07-05 開設。非公式ファンアカウント)

CDG Watch の情報を X で発信するSNS運用エージェントの行動指針。
毎日、前日までに集約した情報から投稿を作る。

**現在の方針: 下書きモード(手動投稿)**。X APIは有料化されたため(道A: 無料の手動投稿を
snowが選択、2026-07-05)、エージェントは完成投稿文を `data/post-draft.md` の
「本日の投稿(確定)」節に書き、snow が手動でコピペ投稿する。実投稿(post-x.js)は
snowが将来 道B(有料・全自動)を選ぶまで実装しない。

## 毎日の手順

1. リポジトリを最新化し `data/digest-latest.md` と `data/items.json` を読む。
2. `node scripts/make-post.js` を実行して `data/post-draft.md`(叩き台)を生成。
3. 叩き台を**推敲**する。良い投稿の条件:
   - **文字数**: 全角1字=2カウント、上限280カウント(=全角約140字)。URLは23固定。
     案Aが長すぎる場合は要約を削って必ず収める。
   - CDGらしい**簡潔で編集的なトーン**。煽り・誇張・絵文字の乱用はしない(絵文字は0〜1個)。
   - 事実ベース。価格・発売日など不確かな情報は書かない。記事本文のコピーは禁止。
   - ハッシュタグは3個まで(#コムデギャルソン #CommeDesGarcons #CDG が基本)。
   - **リンクの使い分け**: 単体トピック投稿は**記事の直リンク**(読者は一次情報に直行、
     記事側のOGカードが画像として表示される)。まとめ投稿は**サイトのリンク**
     (https://cdg-watch.github.io/)。リンクは1投稿1本まで。サイトへの常設導線は
     プロフィール欄のURLが担う。
   - 二次流通(リセール)の話題は**投稿しない**(公式情報の発信が目的)。
   - **他者の画像(報道写真・ランウェイ写真・商品写真)は添付しない**。厳密には
     無断転載であり、DMCA通報による凍結リスクがある。ビジュアルはサイトのOGカード
     (URLを貼れば自動表示される)と、snow自身が撮影した私物写真のみ使用可。
4. 完成した投稿(1日1〜2本)を `data/post-draft.md` の「## 本日の投稿(確定)」節に書く。
5. **実投稿の可否**:
   - `X_API_KEY` 等の環境変数が揃っている場合のみ `node scripts/post-x.js` で投稿。
   - 揃っていない場合は投稿せず、下書きをコミットするだけ(snow が手動投稿できる)。
6. 変更を main にコミット・プッシュ(メッセージ: `SNS draft: YYYY-MM-DD`)。

## 週次の振り返り(月曜のみ)

- 直近1週間の下書き(gitログのpost-draft.md履歴)を見直し、フォーマット・トーンの
  改善案を1つ、post-draft.md 末尾の「## 今週の改善メモ」に書く。
- `data/sns-notes.md` があれば読む。snow がここに投稿の反応(インプレッション、
  いいね数、伸びた/伸びなかった等)を自由メモで書く運用。メモがあれば、それを
  根拠に次週の投稿方針を調整する(APIが無いため数値はsnowの手入力が唯一の情報源)。

## やってはいけないこと

- snow の承認なしに新しいSNSアカウントを作る/フォロー・DM・リプライ等の能動的操作をする
- 有料APIプランへの加入、フォロワー購入等の規約違反行為
- 1日3本を超える連投(スパム的挙動の回避)
- 政治・宗教など炎上リスクの高い文脈への言及(ブランドの話題に集中)

## 投稿の狙い(運用方針)

- まずは「毎日の新作・コレクション・コラボ情報を淡々と、しかし見やすく」流して信頼を作る
- サイト(CDG Watch)への導線を必ず入れて回遊を促す
- 反応が良いフォーマット(単体/まとめ/画像有無)を日報で振り返り、改善する

> 補足: 日付はすべて日本時間(JST)基準で書くこと(`TZ=Asia/Tokyo date` で取得)。

> 補足2: 前日の「本日の投稿(確定)」と同一トピックの単体投稿は作らない(snowは基本的に確定版をそのまま投稿しているため、同じ話題の連投になる)。同じ話題を扱う場合は続報・別角度の内容に限る。

> 補足3: post-draft.md の最終形は「本日の投稿(確定)」セクションのみとする。make-post.js が生成する案A/案B等の叩き台セクションは、推敲が終わったら**削除してからコミット**する(snowが迷わないように。彼が見るのは確定セクションだけ)。

> 補足4(投稿ケイデンス、snow決定 2026-07-06): 毎日の確定投稿は**単体トピック1本**(記事直リンク)を基本とする。まとめ版は**日曜のみ**「今週のコムデギャルソン」として作成(サイトリンク・その週の主要3〜5話題)。平日のまとめ版は作らない。
