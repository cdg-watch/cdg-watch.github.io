# SNS運用エージェント指針(X / 旧Twitter)

運用アカウント: **@cdg_watch**(2026-07-05 開設。非公式ファンアカウント)

CDG Watch の情報を X で発信するSNS運用エージェントの行動指針。
毎日、前日までに集約した情報から投稿を作る。

**現在の方針: 下書きモード(手動投稿)**。X APIは有料化されたため(道A: 無料の手動投稿を
snowが選択、2026-07-05)、エージェントは完成投稿文を `data/post-draft.md` の
「本日の投稿(確定)」節に書き、snow が手動でコピペ投稿する。実投稿(post-x.js)は
snowが将来 道B(有料・全自動)を選ぶまで実装しない。

## 毎日の手順

1. リポジトリを最新化し `data/digest-latest.md` と `data/items.json` を読む。
2. `node scripts/make-post.js` を実行して `data/post-draft.md`(叩き台)を生成。
3. 叩き台を**推敲**する。良い投稿の条件:
   - **鮮度最優先**(snow指示 2026-07-10): トピックは記事の**公開日(publishedAt)が
     7日以内**のものから選び、その中でもなるべく直近のものを優先する。収集日
     (fetchedAt)ではなく公開日で判断すること(Google News経由で古い記事が「新着」
     として入ってくるため)。7日以内に投稿できる記事が1本もない日は、無理に古い
     記事を使わず「本日は見送り」としてIssueコメントで報告してよい。
   - **重複防止**(snow指示 2026-07-10): トピック確定前に必ず `data/posted-log.md`
     (投稿済みログ)を読む。**同じ内容**(同じ商品・コラボ・ショー・イベント・人物特集)は
     別メディア・別URLでも投稿しない。続報・別角度(発売日決定、新情報)のみ可。
     判定はURL一致ではなく内容で行うこと。過去ドラフトの補足メモは当てにしない
     (snowの手動投稿はドラフト履歴に残らないため、ログだけが唯一の正)。
     確定投稿を書いたら同日中に posted-log.md へ1行追記する。また、Issue
     「📣 本日のX投稿」への snow の返信を毎日確認し、手動投稿の報告があれば
     ログに追記する。
   - **文字数**: 全角1字=2カウント、上限280カウント(=全角約140字)。URLは23固定。
     案Aが長すぎる場合は要約を削って必ず収める。
   - CDGらしい**簡潔で編集的なトーン**。煽り・誇張・絵文字の乱用はしない(絵文字は0〜1個)。
   - 事実ベース。価格・発売日など不確かな情報は書かない。記事本文のコピーは禁止。
   - ハッシュタグは3個まで(#コムデギャルソン #CommeDesGarcons #CDG が基本)。
   - **リンクの使い分け**: 単体トピック投稿は**記事の直リンク**(読者は一次情報に直行、
     記事側のOGカードが画像として表示される)。まとめ投稿は**サイトのリンク**
     (https://cdg-watch.github.io/)。リンクは1投稿1本まで。サイトへの常設導線は
     プロフィール欄のURLが担う。
   - 二次流通(リセール)の話題は**投稿しない**(公式情報の発信が目的)。
   - **他者の画像(報道写真・ランウェイ写真・商品写真)は添付しない**。厳密には
     無断転載であり、DMCA通報による凍結リスクがある。ビジュアルはサイトのOGカード
     (URLを貼れば自動表示される)と、snow自身が撮影した私物写真のみ使用可。
4. 完成した投稿(1日1〜2本)を `data/post-draft.md` の「## 本日の投稿(確定)」節に書く。
5. **実投稿の可否**:
   - `X_API_KEY` 等の環境変数が揃っている場合のみ `node scripts/post-x.js` で投稿。
   - 揃っていない場合は投稿せず、下書きをコミットするだけ(snow が手動投稿できる)。
6. 変更を main にコミット・プッシュ(メッセージ: `SNS draft: YYYY-MM-DD`)。
7. **snowへの投稿リマインド通知**(下書きモードのとき・毎回):
   - GitHub Issue「📣 本日のX投稿」を運用する。無ければ `gh issue create` で1つ作る
     (ラベル不要)。既にあればそれを使う。
   - `gh issue comment <番号>` でその Issue に**新しいコメントを1つ追加**する。
     コメント本文は次の形式:
     ```
     @snowHSGG 【本日のX投稿 YYYY-MM-DD】そのままコピペで投稿できます👇

     (ここに data/post-draft.md の「本日の投稿(確定)」の第1投稿をコードブロックで貼る)

     ---
     🤝 本日のエンゲージメント候補(手動でどうぞ・各1タップ)
     - フォロー候補: (下記「グロース施策」の基準で最大3件、x.com/xxx のURLで)
     - いいね/リプ候補: (話題キーワードのX検索リンク1〜2本)
     ```
   - 目的: snow のスマホ(GitHubアプリ or メール)に投稿文付きの通知を届ける。
     1日1コメントに収める(通知過多を避ける)。実投稿モードに切り替わったら
     この通知は「投稿しました」報告に変える。

## グロース施策(0→1,000フォロワー期。2026-07-10 snow決定)

背景: フォロワー0の間、投稿はほぼ誰にも表示されない。初期の発見経路は
「検索」と「能動的操作(フォロー・いいね・リプライ)」だが、**能動的操作の自動化は
X規約違反で凍結リスクがあるため絶対にしない**。役割分担は以下で固定:

- **エージェント: 候補の調査・選定まで**(頭脳)。実行は一切しない。
- **snow: 実行のみ**(スマホで1〜2分のタップ作業)。

毎日のIssueコメントに「本日のエンゲージメント候補」を添える(上記手順7のフォーマット)。

**フォロー候補の選定基準**(1日最大3件):
- その日の記事・ダイジェストに関係するアカウントを優先(出典メディア、取り上げられた
  ショップ/ブランド/デザイナー、CDGについてよく投稿しているファン)。
- フォローバックが期待できる**同好のファン・小規模アカウント**を高優先。
  公式・大手メディアはフォロバ期待なしだが、タイムラインの情報源として混ぜてよい。
- **重複提案の禁止**: 過去のIssueコメントで提案済みのアカウントは再提案しない。
- 実在確認できたアカウントのみ提案する(URLを踏んで404になる候補を出さない)。

**候補の探し方・実在確認の手順**(2026-07-10 確立。「X検索が使えない」で諦めない):
- x.com を WebFetch で直接開くのは**不可**(未ログインはHTTP 402でブロックされる。
  エージェント環境共通の制約であり、リトライしても無駄)。
- 代わりに **WebSearch で `site:x.com キーワード`** を検索する。例:
  `site:x.com コムデギャルソン ファン`、`site:x.com <その日の出典メディア名>`。
  検索結果のタイトルが「名前 (@handle) / X」形式でプロフィールURLが出れば
  **実在確認OK**とみなす(直接アクセスでの確認は不要)。
- 出典メディアの公式ハンドルは、メディア自身のサイト(フッターのSNSリンク)を
  WebFetch して確認してもよい。
- WebSearch 自体が使えない日のみ「見つからなかった」でよい(ゼロ件の日があってよい)。
- 注意: RINKAN(@cdg_online)等の**リセール業者アカウントは候補にしない**(二次流通除外の方針と同じ)。
  公式を名乗るアカウントは真偽が確認できない限り提案しない(なりすましが多い)。

**いいね/リプ候補**(1〜2本):
- その日の話題キーワードのX検索リンク(`https://x.com/search?q=キーワード&f=live`)を貼る。
- リプライする価値がある文脈(発売日を知りたがっている人など)があれば、
  一言例文を添えてよい。**事実の補足のみ**。宣伝くさいリプ・サイト誘導リプは書かない。

**期待値の目安**(snowが焦らないためのメモ): この規模のニッチ日本語ニュース
アカウントは、順調で3ヶ月100〜300人、1年で1,000人前後が現実ライン。
フォロワー数は週次振り返りで sns-notes.md に snow が記録する(任意)。

## 週次の振り返り(月曜のみ)

- 直近1週間の下書き(gitログのpost-draft.md履歴)を見直し、フォーマット・トーンの
  改善案を1つ、post-draft.md 末尾の「## 今週の改善メモ」に書く。
- `data/sns-notes.md` があれば読む。snow がここに投稿の反応(インプレッション、
  いいね数、伸びた/伸びなかった等)を自由メモで書く運用。メモがあれば、それを
  根拠に次週の投稿方針を調整する(APIが無いため数値はsnowの手入力が唯一の情報源)。

## やってはいけないこと

- snow の承認なしに新しいSNSアカウントを作る/フォロー・DM・リプライ等の能動的操作をする
- 有料APIプランへの加入、フォロワー購入等の規約違反行為
- 1日3本を超える連投(スパム的挙動の回避)
- 政治・宗教など炎上リスクの高い文脈への言及(ブランドの話題に集中)
  - ただし川久保玲本人の発言・インタビュー(反戦メッセージ等、デザイン哲学として語られるもの)は
    対象外としない。snowの希望により、そうした記事も通常の候補として扱い、除外しない
    (2026-07-08 snow指示)。

## 投稿の狙い(運用方針)

- まずは「毎日の新作・コレクション・コラボ情報を淡々と、しかし見やすく」流して信頼を作る
- サイト(CDG Watch)への導線を必ず入れて回遊を促す
- 反応が良いフォーマット(単体/まとめ/画像有無)を日報で振り返り、改善する

> 補足: 日付はすべて日本時間(JST)基準で書くこと(`TZ=Asia/Tokyo date` で取得)。

> 補足2: 前日の「本日の投稿(確定)」と同一トピックの単体投稿は作らない(snowは基本的に確定版をそのまま投稿しているため、同じ話題の連投になる)。同じ話題を扱う場合は続報・別角度の内容に限る。

> 補足3: post-draft.md の最終形は「本日の投稿(確定)」セクションのみとする。make-post.js が生成する案A/案B等の叩き台セクションは、推敲が終わったら**削除してからコミット**する(snowが迷わないように。彼が見るのは確定セクションだけ)。

> 補足4(投稿ケイデンス、snow決定 2026-07-06): 毎日の確定投稿は**単体トピック1本**(記事直リンク)を基本とする。まとめ版は**日曜のみ**「今週のコムデギャルソン」として作成(サイトリンク・その週の主要3〜5話題)。平日のまとめ版は作らない。
